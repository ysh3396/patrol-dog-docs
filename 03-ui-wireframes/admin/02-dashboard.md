## 레이아웃 설명

### KPI 카드 (4개, 상단 가로 배치)

| 카드 | 데이터 소스 | 비교 기준 |
|------|------------|-----------|
| 총 대원 수 | User 전체 count (role=user) | 전주 대비 증감률 |
| 금일 순찰 건수 | Patrol (status=completed, 오늘) count | 전일 대비 증감률 |
| 미처리 신고 | Report (status=submitted) count | 전주 대비 증감률 |
| 금주 신규 가입 | User (이번 주 가입) count | 전주 대비 증감률 |

### 차트 영역 (2열 배치)

| 차트 | 유형 | 데이터 |
|------|------|--------|
| 일별 순찰 추이 | 라인 차트 | 최근 7일간 Patrol(completed) 일별 건수 |
| 지역별 순찰 분포 | 수평 바 차트 | 지역(구/군)별 Patrol 건수 상위 6개 |

### 최근 활동 목록

- 최근 순찰 완료, 신고 접수, 신규 가입 등 5건 시간순 표시
- Report.status 값은 glossary 기준 (`submitted`, `in_progress`, `resolved`) 표시

## 인터랙션 설명

| 요소 | 동작 | 결과 |
|------|------|------|
| "총 대원 수" 카드 클릭 | 페이지 이동 | A02 대원 관리 (목록) |
| "미처리 신고" 카드 클릭 | 페이지 이동 + 필터 | A03 신고 관리 (submitted 필터 적용) |
| "금일 순찰 건수" 카드 클릭 | 페이지 이동 | A07 통계 (오늘 기준) |
| 최근 활동 행 클릭 | 해당 상세 이동 | 순찰: A07, 신고: A03 상세, 가입: A02 상세 |
| 사이드바 메뉴 클릭 | 해당 화면 이동 | A01~A10 각 화면 |
| "로그아웃" 클릭 | 세션 종료 | A09 로그인 화면 이동 |
| 관리자명 ▼ 클릭 | 드롭다운 | 내 정보, 로그아웃 옵션 |

## 데이터 갱신

- KPI 카드: 페이지 로드 시 자동 조회, 60초 간격 자동 갱신
- 차트: 페이지 로드 시 1회 조회
- 최근 활동: 페이지 로드 시 조회, 30초 간격 자동 갱신
