## 레이아웃 설명

- **전체 화면 (사이드바 없음)**: 로그인 전이므로 사이드바/헤더 미표시
- **가운데 정렬 로그인 폼**: 화면 중앙에 카드 형태로 배치
- **앱 로고 + 타이틀**: 카드 상단에 순찰대 로고와 "Admin Console" 텍스트
- **푸터**: 하단에 저작권 표시

---

## 인터랙션 설명

| 요소 | 동작 | 결과 |
|------|------|------|
| 이메일 입력 | 텍스트 입력 | 이메일 형식 실시간 검증 |
| 비밀번호 입력 | 텍스트 입력 | 마스킹 처리 (●●●) |
| [로그인] 버튼 클릭 | 서버 인증 요청 | 아래 분기 참조 |
| Enter 키 | [로그인] 버튼과 동일 | 폼 제출 |

### 로그인 결과 분기

| 조건 | 결과 | 이동 화면 |
|------|------|-----------|
| 인증 성공 + role=admin | 세션 생성, 대시보드 이동 | A01 대시보드 |
| 인증 실패 (잘못된 비밀번호) | 에러 메시지 표시: "비밀번호가 일치하지 않습니다" | A09 유지 |
| 인증 실패 (미등록 이메일) | 에러 메시지 표시: "등록되지 않은 이메일입니다" | A09 유지 |
| 인증 성공 + role=user | 에러 메시지 표시: "관리자 권한이 없습니다" | A09 유지 |
| 필드 미입력 | 해당 필드 하단 에러: "필수 항목입니다" | A09 유지 |

## 에러 메시지 표시 규칙

- 에러 메시지는 [로그인] 버튼 하단에 빨간색 텍스트로 표시
- 필드별 유효성 에러는 각 입력 필드 하단에 표시
- 에러 발생 시 비밀번호 필드 초기화, 이메일 필드 유지
