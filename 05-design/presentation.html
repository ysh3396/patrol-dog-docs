SECTION 1 · 프로젝트 소개

프로젝트 배경

🏙️

도시 안전 사각지대 증가

골목길·야간 취약 지역이 늘어나는 가운데 지역 방범 활동의 디지털화가 필요한 시점입니다. 기존 방범 활동은 종이 기록·수기 출석 체크에 머물러 있어 데이터 활용이 불가했습니다.

🐕‍🦺

일상 산책 → 방범 순찰로 전환

반려견주는 매일 1~2회 산책을 하며, 이 경로가 곧 지역 방범 순찰 경로가 될 수 있습니다. GPS 기반 트래킹으로 산책을 지역 안전 활동으로 자연스럽게 연결합니다.

📊

활동 데이터 체계화 부재

순찰 거리·시간·횟수를 자동 집계하는 도구가 없어 활동량을 파악하거나 우수 대원을 객관적으로 평가하기 어렵습니다. 데이터 기반 운영이 필요합니다.

💡

반려견순찰대 앱의 사회적 가치

반려견 산책이라는 일상을 지역 안전 활동으로 전환하는 플랫폼 — 개인의 건강관리, 동네 방범, 커뮤니티 소속감을 하나의 앱으로 통합합니다.

SECTION 1 · 프로젝트 소개

프로젝트 목적

1

실시간 GPS 트래킹

반려견 산책 및 순찰 경로를 실시간으로 기록하고 지도 위에 시각화합니다.

2

방범 활동 기록 / 통계화

순찰 거리·시간·횟수를 자동 집계하여 개인/팀/지역 단위 통계를 제공합니다.

3

대원 커뮤니티

활동 인증·정보 공유·자유 게시판으로 대원 간 소통과 동기부여를 촉진합니다.

4

순찰대 활동 지원 + 게이미피케이션

현장 마킹·신고·관리자 대시보드로 운영을 체계화하고, 포인트·랭킹으로 지속적인 참여를 유도합니다.

GPS 백그라운드 트래킹 전략

iOS — Background Modes + CLLocationManager
핵심 설정
pausesLocationUpdatesAutomatically = false
→ true면 iOS가 임의로 수집 중단 (가장 흔한 이슈)
OS 강제 종료 대응
significantLocationChange로 앱 재시작 트리거 → 트래킹 자동 복원
App Store 심사 전략
showsBackgroundLocationIndicator = true (파란 바 표시) + 명확한 사유 설명
🤖 Android — Foreground Service
핵심 메커니즘
startForeground() → 상단 알림 상시 표시 → OS가 앱을 "사용 중"으로 분류 → 강제 종료 방지
버전별 대응
Android 10+: ACCESS_BACKGROUND_LOCATION
Android 12+: foregroundServiceType="location"
Android 14+: 정확한 위치 권한 UX
