# 03. 기능 명세

> **문서 목적**: 앱의 모든 기능을 영역별로 정의하고, 우선순위(P0/P1/P2)를 부여하여 개발 로드맵의 기준을 제공한다.
> **대상 독자**: 기획자, UX 디자이너, 개발자, PM
> **참조**: 용어 정의는 `06-glossary.md`, 화면 번호 매핑은 `04-information-architecture.md` 참조

## 우선순위 정의

| 등급 | 의미 | 기준 |
|------|------|------|
| **P0** | 필수 (Must Have) | MVP 런칭에 반드시 포함. 이 기능 없이는 서비스가 성립하지 않음 |
| **P1** | 중요 (Should Have) | 런칭 시 포함을 목표로 하되, 일정에 따라 빠른 후속 업데이트로 대체 가능 |
| **P2** | 희망 (Nice to Have) | 런칭 후 사용자 피드백을 반영하여 추가 |

## A. 순찰(트래킹) 시스템

### A-1. GPS 실시간 트래킹 `P0`

| 항목 | 내용 |
|------|------|
| **화면** | U02 순찰 대기, U03 순찰 진행 중 |
| **설명** | 사용자가 순찰을 시작하면 GPS 좌표를 실시간으로 수집하여 지도 위에 경로를 표시한다. |
| **상세 요구사항** | |
| - 시작/종료 | 순찰 대기 화면에서 '시작' 버튼 탭 → 트래킹 시작. '종료' 버튼 탭 → 트래킹 종료 |
| - 경로 표시 | 지도 위에 폴리라인으로 이동 경로를 실시간 렌더링 |
| - 거리 계산 | 수집된 좌표 간 거리를 누적 합산하여 실시간 표시 (단위: km, 소수점 2자리) |
| - 시간 표시 | 순찰 시작 후 경과 시간을 타이머로 표시 (HH:MM:SS) |
| - 현재 속도 | 현재 이동 속도 표시 (km/h) |
| **수용 기준** | 순찰 시작 후 지도 위에 경로가 그려지고, 종료 시 총 거리/시간이 정확히 기록된다. |

### A-2. 백그라운드 실행 `P0`

| 항목 | 내용 |
|------|------|
| **화면** | 시스템 레벨 (UI 없음, 상태바 표시) |
| **설명** | 앱이 백그라운드로 전환되어도 GPS 수집이 끊기지 않는다. |
| **상세 요구사항** | |
| - 위치 수집 조건 | 5~10m 이동 시 또는 10초 간격 중 먼저 충족되는 조건에서 좌표 기록 |
| - 상태바 표시 | 순찰 진행 중 상태바에 위치 사용 중 아이콘 표시 |
| - 배터리 최적화 | iOS: Background Location Updates, Android: Foreground Service 사용 |
| - 연결 끊김 대응 | GPS 신호 유실 시 마지막 유효 좌표 유지, 복구 시 자동 이어서 기록 |
| **수용 기준** | 앱 전환 후 5분간 이동한 경로가 누락 없이 기록된다. |

### A-3. 현장 기록 (마킹) `P1`

| 항목 | 내용 |
|------|------|
| **화면** | U05 현장 마킹 |
| **설명** | 순찰 중 특정 위치에서 사진 촬영 + 메모를 남겨 현장 기록을 생성한다. |
| **상세 요구사항** | |
| - 사진 촬영 | 카메라 직접 촬영 또는 갤러리에서 선택 (최대 3장) |
| - 위치 자동 기록 | 마킹 시점의 GPS 좌표 자동 저장 |
| - 메모 입력 | 텍스트 메모 입력 (최대 200자) |
| - 유형 선택 | 일반 기록 / 신고 중 선택 |
| - 지도 표시 | 마킹된 위치를 경로 위에 핀으로 표시 |
| **수용 기준** | 순찰 중 마킹 생성 시 사진, 위치, 메모가 저장되고 경로 지도에 핀이 표시된다. |

### A-4. 활동 일지 `P1`

| 항목 | 내용 |
|------|------|
| **화면** | U07 순찰 결과 |
| **설명** | 순찰 종료 시 결과 요약과 함께 활동 일지(메모)를 입력할 수 있다. |
| **상세 요구사항** | |
| - 결과 요약 자동 표시 | 총 거리, 총 시간, 마킹 수, 획득 포인트 |
| - 일지 입력 | 순찰 소감/특이사항 텍스트 입력 (선택, 최대 500자) |
| - 저장 | 일지 미입력 시에도 순찰 기록은 자동 저장 |
| **수용 기준** | 순찰 종료 후 결과 요약이 표시되고, 일지 입력 후 저장이 완료된다. |

### A-5. 순찰 일시정지/재개 `P1`

| 항목 | 내용 |
|------|------|
| **화면** | U04 순찰 일시정지 |
| **설명** | 순찰 중 일시정지하여 GPS 수집을 중단하고, 재개 시 이어서 기록한다. |
| **상세 요구사항** | |
| - 일시정지 | 정지 버튼 탭 → GPS 수집 중단, 타이머 정지 |
| - 재개 | 재개 버튼 탭 → GPS 수집 재시작, 타이머 재시작 |
| - 정지 중 표시 | 지도에 정지 구간을 점선으로 표시 |
| - 정지 시간 제한 | 30분 이상 정지 시 자동 종료 안내 팝업 |
| **수용 기준** | 일시정지 후 재개 시 경로가 이어서 기록되고, 정지 시간은 총 순찰 시간에서 제외된다. |

## B. 포인트 및 게이미피케이션

### B-1. 포인트 시스템 `P0`

| 항목 | 내용 |
|------|------|
| **화면** | U07 순찰 결과, U18 포인트 내역 |
| **설명** | 순찰 거리에 비례하여 포인트를 자동 지급한다. |
| **상세 요구사항** | |
| - 기본 공식 | 1km = 100포인트 (확정은 운영 정책에 따라 조정 가능) |
| - 소수점 처리 | 0.1km 단위로 비례 계산 (예: 2.3km = 230pt) |
| - 최소 거리 | 0.1km 미만 순찰은 포인트 미지급 |
| - 일일 상한 | 1일 최대 1,000포인트 (어뷰징 방지, 운영 정책 조정 가능) |
| - 보너스 포인트 | 이벤트 기간 배율 적용 가능 (관리자 설정) |
| - 포인트 내역 | 획득/차감 이력을 날짜별로 조회 가능 |
| **수용 기준** | 순찰 종료 시 거리에 비례한 포인트가 자동 지급되고, 포인트 내역에 기록된다. |

### B-2. 랭킹 시스템 `P1`

| 항목 | 내용 |
|------|------|
| **화면** | U14 랭킹 (리더보드) |
| **설명** | 사용자들의 순찰 활동을 기준으로 랭킹을 산출하여 선의의 경쟁을 유도한다. |
| **상세 요구사항** | |
| - 기간별 탭 | 실시간 / 주간 / 월간 3개 탭 |
| - 랭킹 기준 | 누적 순찰 거리 (km) 기준 정렬 |
| - 표시 항목 | 순위, 프로필 이미지, 닉네임, 총 거리, 총 포인트 |
| - 내 순위 강조 | 리더보드에서 내 위치를 하이라이트 표시 |
| - 갱신 주기 | 실시간 탭은 순찰 종료 시 즉시 반영, 주간/월간은 매일 자정 갱신 |
| **수용 기준** | 랭킹 화면에서 기간별 리더보드가 표시되고, 내 순위가 정확하다. |

### B-3. 어뷰징 방지 `P1`

| 항목 | 내용 |
|------|------|
| **화면** | 시스템 레벨 (백엔드 검증) |
| **설명** | 비정상적인 이동 속도를 감지하여 부정 활동을 방지한다. |
| **상세 요구사항** | |
| - 도보 임계값 | 평균 속도 6km/h 초과 시 경고 (확정은 베타 테스트 후 조정) |
| - 자전거 임계값 | 평균 속도 25km/h 초과 시 경고 (확정은 베타 테스트 후 조정) |
| - 차량 감지 | 40km/h 이상 지속 시 해당 구간 포인트 미지급 |
| - 경고 처리 | 1차: 경고 알림, 2차: 해당 순찰 포인트 미지급, 3차: 관리자 통보 |
| - GPS 조작 감지 | 위치 점프(순간이동) 감지 시 해당 구간 무효 처리 |
| **Open Questions** | 정확한 임계값은 베타 테스트 데이터를 수집한 후 확정. 자전거 순찰 허용 여부도 운영 정책에서 결정. |
| **수용 기준** | 차량 이동으로 판단되는 속도에서는 포인트가 지급되지 않는다. |
