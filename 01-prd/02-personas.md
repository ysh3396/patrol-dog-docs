# 02. 페르소나

> **문서 목적**: 타겟 사용자를 구체적인 인물로 형상화하여 기획/디자인 의사결정의 기준점을 제공한다.
> **대상 독자**: 기획자, UX 디자이너, 개발자

---

## 페르소나 1: 김지현 — 반려견주

| 항목 | 내용 |
|------|------|
| **이름** | 김지현 |
| **나이** | 34세 |
| **직업** | IT 회사 마케터 |
| **거주지** | 서울 마포구 |
| **반려견** | 말티즈 '콩이' (3살) |

### 목표
- 콩이와 매일 안전하게 산책하고 싶다.
- 산책 경로와 거리를 기록해서 건강 관리에 활용하고 싶다.
- 동네 안전에 기여하고 있다는 소속감을 느끼고 싶다.

### 고충 (Pain Points)
- 어두운 시간대에 산책할 때 으슥한 골목이 불안하다.
- 산책 기록을 별도 앱(건강 앱)에서 관리하는 게 번거롭다.
- 동네에서 비슷한 시간대에 산책하는 사람들이 있는지 알 수 없다.

### 사용 시나리오
> 퇴근 후 저녁 8시, 지현이는 콩이 목줄을 채우고 앱을 켠다. **[U02 순찰 대기]** 화면에서 '순찰 시작' 버튼을 누르고 평소 루트로 산책을 시작한다. 공원 벤치 근처에 파손된 가로등을 발견하고 **[U05 현장 마킹]**으로 사진을 찍어 기록한다. 30분 후 집에 도착해 순찰을 종료하면 **[U07 순찰 결과]**에서 2.1km를 걸었고 210포인트를 획득했다는 결과를 확인한다. **[U10 피드]**에 오늘의 산책 사진을 올리고, 같은 동네 대원들의 반응을 기대하며 앱을 닫는다.

## 페르소나 2: 박성호 — 방범대원

| 항목 | 내용 |
|------|------|
| **이름** | 박성호 |
| **나이** | 52세 |
| **직업** | 자영업 (철물점 운영) |
| **거주지** | 경기도 수원시 |
| **반려견** | 진돗개 '바둑이' (5살) |

### 목표
- 지역 방범 순찰 활동을 체계적으로 기록하고 인정받고 싶다.
- 순찰 거리와 횟수로 다른 대원들과 선의의 경쟁을 하고 싶다.
- 현장에서 발견한 이상 징후를 간편하게 신고하고 싶다.

### 고충 (Pain Points)
- 현재 순찰 기록은 종이 일지에 수기로 작성하고 있어 번거롭다.
- 얼마나 걸었는지, 어느 구역을 순찰했는지 정확히 파악하기 어렵다.
- 순찰 중 이상 상황을 발견해도 사진과 위치를 함께 기록할 도구가 없다.
- 활동량이 많아도 별다른 보상이나 인정 체계가 없다.

### 사용 시나리오
> 토요일 오전 7시, 성호 씨는 바둑이와 함께 동네 순찰에 나선다. **[U02 순찰 대기]**에서 순찰을 시작하고, 아파트 단지 뒤편 골목을 돌기 시작한다. 비닐하우스 뒤에 쓰레기가 불법 투기된 현장을 발견하고 **[U05 현장 마킹]**으로 사진 촬영 후 **[U06 신고 작성]**으로 신고를 접수한다. 1시간 30분 동안 5.2km를 순찰하고 종료한다. **[U14 랭킹]**에서 이번 주 3위에 올랐다는 것을 확인하며 뿌듯해한다. 월말에 관리자가 보내주는 활동 통계에서 이번 달 총 42km를 순찰했다는 기록을 본다.

## 페르소나 3: 이수진 — 관리자

| 항목 | 내용 |
|------|------|
| **이름** | 이수진 |
| **나이** | 41세 |
| **직업** | 수원시 자치행정과 주무관 |
| **거주지** | 경기도 수원시 |
| **반려견** | 없음 |

### 목표
- 관할 지역 방범대원들의 활동을 한눈에 파악하고 싶다.
- 월간/분기별 순찰 통계를 보고서로 추출하고 싶다.
- 신고 내역을 체계적으로 관리하고 처리 상태를 추적하고 싶다.
- 이벤트와 공지를 효율적으로 전달하고 싶다.

### 고충 (Pain Points)
- 대원별 활동량을 일일이 수집하여 엑셀로 정리하는 데 시간이 많이 든다.
- 신고 접수 후 처리 현황을 추적할 시스템이 없어 민원이 발생한다.
- 공지사항을 카카오톡 단체방으로 전달하면 읽었는지 확인이 어렵다.
- 우수 대원 포상 기준이 주관적이라 불만이 생긴다.

### 사용 시나리오
> 월요일 오전, 수진 씨는 **[A01 관리자 대시보드]**에 접속하여 지난주 순찰 통계를 확인한다. 총 87건의 순찰이 진행되었고 누적 거리는 312km이다. **[A03 신고 관리]**에서 미처리 신고 3건을 확인하고 담당 부서에 전달한다. 월말이 다가와 **[A07 통계]**에서 이번 달 대원별 활동 데이터를 CSV로 추출하여 보고서를 작성한다. 다가오는 '봄맞이 안전 순찰 이벤트'를 **[A04 이벤트 관리]**에서 등록하고, **[A06 Push 알림 관리]**를 통해 전체 대원에게 참여 안내를 발송한다.

## 페르소나 4: 최민서 — 일반 산책러

| 항목 | 내용 |
|------|------|
| **이름** | 최민서 |
| **나이** | 26세 |
| **직업** | 대학원생 |
| **거주지** | 서울 관악구 |
| **반려견** | 없음 (동네 고양이는 좋아함) |

### 목표
- 건강을 위해 매일 걷기 운동을 하면서 기록하고 싶다.
- 포인트와 랭킹으로 동기부여를 받고 싶다.
- 가볍게 동네 안전에도 기여하고 싶지만 부담은 싫다.

### 고충 (Pain Points)
- 기존 걷기 앱은 단조로워서 동기부여가 되지 않는다.
- 혼자 걷다 보면 금세 질려서 꾸준히 하기 어렵다.
- 방범 순찰이라고 하면 너무 무겁고 부담스럽게 느껴진다.
- SNS처럼 활동을 공유하고 반응을 받을 수 있으면 좋겠다.

### 사용 시나리오
> 점심시간, 민서는 캠퍼스 주변을 산책하면서 앱을 켠다. **[U02 순찰 대기]**에서 가볍게 순찰을 시작한다. 30분 동안 1.5km를 걸으며 벚꽃이 예쁜 길을 **[U05 현장 마킹]**으로 사진 촬영한다. 순찰 종료 후 150포인트를 획득하고, **[U14 랭킹]**에서 이번 주 캠퍼스 근처 순찰 거리가 7위라는 것을 확인한다. **[U10 피드]**의 활동인증 게시판에 벚꽃 사진과 함께 "오늘도 1.5km 완주!"라고 올린다. 다른 사용자들의 좋아요와 댓글에 뿌듯함을 느끼며 내일도 걸어야겠다고 다짐한다.
