# 03. 화면 전환 다이어그램 (Screen Transition Diagram)

> **문서 목적**: 모바일 앱(U00~U23)과 관리자 웹(A01~A10) 전체 34개 화면의 전환 관계를 정의하여 네비게이션 설계의 기준을 제공한다.
> **대상 독자**: UX 디자이너, 프론트엔드 개발자, 기획자
> **참조**: 화면 번호는 `04-information-architecture.md`, 용어는 `06-glossary.md` 기준

---

## 1. 모바일 앱 전체 화면 전환도

### 1.1 온보딩 → 메인 진입

```
[U00 스플래시]
    │
    ├── 최초 실행 ──▶ [U00a 온보딩 슬라이드]
    │                    │
    │                    ├── 슬라이드 완료 / "건너뛰기"
    │                    │
    │                    ▼
    │               [U00b 로그인]
    │
    ├── 재실행 + 토큰 만료 ──▶ [U00b 로그인]
    │
    └── 재실행 + 토큰 유효 ──▶ [U02 순찰 대기 (홈 탭)]
```

### 1.2 로그인/가입/비밀번호 찾기

```
[U00b 로그인]
    │
    ├── 로그인 성공 ──────────────────────────▶ [U02 순찰 대기 (홈 탭)]
    │
    ├── "회원가입" 링크 ──▶ [U00c 회원가입]
    │                          │
    │                          ├── 가입 성공 ──▶ [U02 순찰 대기 (홈 탭)]
    │                          │
    │                          └── "뒤로" ──▶ [U00b 로그인]
    │
    └── "비밀번호 찾기" 링크 ──▶ [U00d 비밀번호 찾기]
                                    │
                                    ├── 재설정 완료 ──▶ [U00b 로그인]
                                    │
                                    └── "뒤로" ──▶ [U00b 로그인]
```

### 1.3 메인 탭바 구조

```
┌─────────────────────────────────────────────────────────────┐
│                        [헤더 영역]                           │
│   ← 뒤로(스택 있을 때)    앱 로고/타이틀    🔔 알림(U09)     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                     [콘텐츠 영역]                             │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [🏠 홈]     [💬 커뮤니티]     [🏆 랭킹]     [👤 마이페이지]  │
│   (탭1)        (탭2)          (탭3)         (탭4)           │
└─────────────────────────────────────────────────────────────┘

탭 전환 (수평 이동, 스택 초기화):
  [탭1 홈] ⟷ [탭2 커뮤니티] ⟷ [탭3 랭킹] ⟷ [탭4 마이페이지]
```

### 1.8 공통: 알림

```
[모든 화면 헤더] ── 🔔 벨 아이콘 탭
    │
    ▼
[U09 알림 목록]
    │
    ├── [알림 항목 탭] ──▶ 딥링크 대상 화면으로 이동
    │                       │
```

### 1.9 공통: 권한 요청 팝업

```
[U21 위치 권한 요청] ── 시스템 팝업 (모달)
    │
    ├── "허용" ──▶ 요청한 기능 진행
    │
    ├── "거부" ──▶ 기능 제한 안내 + 설정 이동 유도
    │
    └── "영구 거부" ──▶ OS 설정 이동 딥링크


[U22 카메라/갤러리 권한] ── 시스템 팝업 (모달)
    │
    ├── "허용" ──▶ 카메라/갤러리 접근
    │
    └── "거부" ──▶ 기능 제한 안내
```

### 1.10 공통: 에러 화면

```
[U23 에러 화면]
    │
    ├── 네트워크 오류:
    │   ├── "인터넷 연결을 확인해주세요"
    │   └── [재시도] ──▶ 이전 요청 재실행 → 성공 시 원래 화면
    │
    └── 서버 오류:
        ├── "일시적인 오류가 발생했습니다"
        └── [재시도] ──▶ 이전 요청 재실행
```

## 3. 관리자 웹 전체 화면 전환도

### 3.1 로그인 → 대시보드

```
[A09 로그인]
    │
    ├── 로그인 성공 (role='admin') ──▶ [A01 대시보드]
    │
    └── 로그인 실패 ──▶ [A09 로그인] (에러 메시지)
```

### 3.2 사이드바 네비게이션 기반 전환

```
┌─────────────────────────────────────────────────────────┐

│  ├──────────────┤  │     ├── 총 대원(User) 수            │
│  │ A02 대원 관리  │──┼──▶ 대원 목록/상세                  │
│  ├──────────────┤  │     ├── 미처리 Report 건수 클릭      │
│  │ A03 신고 관리  │──┼──▶ 신고 목록/상세/상태 변경          │
```

## 4. 전환 유형 분류

### 4.1 네비게이션 유형

| 유형 | 설명 | 적용 화면 | 전환 효과 |
|------|------|----------|----------|
| **탭 전환** | 하단 탭바로 루트 화면 간 이동 | U02, U10, U14, U15 | 페이드 또는 즉시 전환, 스택 초기화 |
| **Push (스택)** | 현재 화면 위에 새 화면을 쌓음 | 대부분의 depth 이동 | 오른쪽에서 슬라이드 인 |
| **Pop (뒤로)** | 스택에서 현재 화면 제거, 이전 화면 노출 | "뒤로" 버튼, 스와이프 백 | 오른쪽으로 슬라이드 아웃 |
| **모달 (바텀시트)** | 현재 화면 위에 반투명 오버레이 | 권한 요청, 확인 팝업, 필터 | 아래에서 슬라이드 업 |
| **Replace (교체)** | 현재 화면을 새 화면으로 교체 (스택 유지 안 함) | 로그인 성공→홈, 로그아웃→로그인 | 페이드 전환 |

## 5. 딥링크 전환 경로

> PushNotification 탭 또는 외부 링크를 통해 앱 내 특정 화면으로 직접 이동하는 경로.

```
[Push 알림 탭 / 외부 딥링크]
    │
    ├── 앱 미실행
    │     │
    │     ▼
    │   [U00 스플래시] → 토큰 확인
    │     │
    │     ├── 토큰 유효 ──▶ [딥링크 대상 화면]
    │     │
    │     └── 토큰 만료 ──▶ [U00b 로그인] → 로그인 성공 ──▶ [딥링크 대상 화면]
    │
    └── 앱 실행 중
          │
          ▼
        [현재 화면] → [딥링크 대상 화면] (Push 전환)
          └── "뒤로" ──▶ [현재 화면으로 복귀]


딥링크 대상 화면 매핑:
  ├── patrol/{id}     ──▶ [U08 순찰 기록 상세]
  ├── post/{id}       ──▶ [U11 게시글 상세]
  ├── ranking         ──▶ [U14 랭킹]
  ├── notifications   ──▶ [U09 알림 목록]
  └── home            ──▶ [U02 순찰 대기]
```

## 6. 엣지 케이스 전환 처리

```
[U03 순찰 진행 중] ── 탭바 다른 탭 탭
    │
    ├── 옵션 A (권장): 탭 전환 허용, 상단 배너 "순찰 진행 중" 표시
    │                    └── 배너 탭 ──▶ [U03 순찰 진행 중] 복귀
    │
    └── 옵션 B: 탭 전환 차단, "순찰을 먼저 종료해주세요" 토스트
```

### 6.2 네트워크 오류 시 화면 전환

```
[서버 요청 필요한 화면 전환 시도] ── 네트워크 오류
    │
    ├── 읽기 요청 (목록 조회 등)
    │     └── [U23 에러 화면] 또는 캐시된 데이터 표시 + 오프라인 배너

          └── 에러 토스트 + 현재 화면 유지 (입력 데이터 보존)
              └── [재시도] 버튼
```

### 6.3 미인증 상태에서 인증 필요 화면 접근

```
    "로그인이 만료되었습니다. 다시 로그인해주세요."
    │
    └── [확인] ──▶ [U00b 로그인]
                     │
                     └── 로그인 성공 ──▶ [원래 접근하려던 화면] (가능한 경우)
```
