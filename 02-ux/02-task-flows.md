# 02. 태스크 플로우 (Task Flows)

> **문서 목적**: 핵심 기능별 상세 태스크 플로우를 정의하여 화면 설계 및 개발의 기준을 제공한다.

## 1. 회원가입 / 로그인 / 비밀번호 찾기

### 1.1 앱 실행 → 로그인 분기

```
[앱 실행]
    │
    ▼
[U00 스플래시] ── 로고 표시 (2초)
    │
    ├── 로그인 토큰 유효? ─── YES ──▶ [U02 순찰 대기 (홈)]
    │
    └── NO (최초 실행 여부 확인)
         │
         ├── 최초 실행 ──▶ [U00a 온보딩 슬라이드]
         │                    │
         │                    ├── 슬라이드 3장 스와이프
         │                    ├── "건너뛰기" 버튼 탭
         │                    │
         │                    ▼
         │               [U00b 로그인]
         │
         └── 재실행 ──▶ [U00b 로그인]
```

### 1.2 이메일 로그인

```
[U00b 로그인]
    │
    ├── 이메일 + 비밀번호 입력
    │     │
    │     ▼
    │   [로그인 버튼 탭]
    │     │
    │     ├── 성공 ──▶ [U02 순찰 대기 (홈)]
    │     │
    │     ├── 실패: 잘못된 비밀번호
    │     │     └── 에러 메시지: "비밀번호가 일치하지 않습니다"
    │     │         └── 재입력 또는 [U00d 비밀번호 찾기]로 이동
    │     │
    │     └── 실패: 미가입 이메일
    │           └── 에러 메시지: "등록되지 않은 이메일입니다"
    │               └── [U00c 회원가입]으로 이동 유도
    │
    └── 소셜 로그인 버튼 탭 (카카오/네이버/Google/Apple)
          │
          ▼
        [소셜 인증 웹뷰/시스템 시트]
          │
          ├── 인증 성공 + 기존 User ──▶ [U02 순찰 대기 (홈)]
          │
          ├── 인증 성공 + 신규 User ──▶ [U00c 회원가입 (닉네임/휴대폰만 입력)]
          │
          └── 인증 취소/실패 ──▶ [U00b 로그인] (에러 토스트)
```

### 1.3 회원가입

```
[U00c 회원가입]
    │
    ├── 입력 항목:
    │   ├── 이메일 ── 실시간 중복 검사
    │   ├── 비밀번호 ── 8자 이상, 영문+숫자+특수문자
    │   ├── 비밀번호 확인 ── 일치 여부 실시간 검사
    │   ├── 닉네임 ── 실시간 중복 검사, 2~10자
    │   └── 휴대폰 번호 ── 형식 검사
    │
    ├── 약관 동의 체크 (필수: 이용약관, 개인정보 처리방침)
    │
    ▼
  [가입 버튼 탭]
    │
    ├── 유효성 통과 ──▶ User 생성 (role='user', points=0)
    │                     │
    │                     ▼
    │                   [가입 완료 토스트]
    │                     │
    │                     ▼
    │                   [U02 순찰 대기 (홈)]
    │
    └── 유효성 실패 ──▶ 해당 필드 하단에 에러 메시지 표시
                         └── 재입력
```

### 1.4 비밀번호 찾기

```
[U00d 비밀번호 찾기]
    │
    ├── 이메일 입력
    │
    ▼
  [재설정 링크 발송 버튼 탭]
    │
    ├── 등록된 이메일 ──▶ "비밀번호 재설정 이메일을 발송했습니다"
    │                       │
    │                       ▼
    │                     [이메일 확인 → 재설정 링크 클릭]
    │                       │
    │                       ▼
    │                     [비밀번호 재설정 웹 페이지]
    │                       │
    │                       ├── 새 비밀번호 입력 + 확인
    │                       │
    │                       ▼
    │                     [변경 완료 → 앱 U00b 로그인으로 이동]
    │
    └── 미등록 이메일 ──▶ "등록되지 않은 이메일입니다"
                           └── 재입력 또는 [U00c 회원가입]으로 이동
```

## 2. Patrol(순찰): 시작 → 일시정지 → 재개 → PatrolMark(마킹) → Report(신고) → 종료

### 2.1 Patrol 시작

```
[U02 순찰 대기]
    │
    ├── 현재 위치 지도 표시 (GPS 활성 상태 확인)
    │
    ▼
  ["순찰 시작" 버튼 탭]
    │
    ├── GPS 권한 확인
    │     │
    │     ├── 권한 허용 상태 ──▶ Patrol 레코드 생성 (status='ready' → 'active')
    │     │                       │
    │     │                       ▼
    │     │                     [U03 순찰 진행 중]
    │     │                       ├── 실시간 경로(Route) 렌더링
    │     │                       ├── 타이머 시작 (HH:MM:SS)
    │     │                       ├── 거리 누적 표시 (km)
    │     │                       └── 현재 속도 표시 (km/h)
    │     │
    │     ├── 권한 미요청 상태 ──▶ [U21 위치 권한 요청 팝업]
    │     │     │
    │     │     ├── "허용" ──▶ Patrol 시작 (위와 동일)
    │     │     └── "거부" ──▶ [설정 안내 팝업]
    │     │                     "순찰을 위해 위치 권한이 필요합니다.
    │     │                      설정에서 위치 권한을 허용해주세요."
    │     │                     [설정으로 이동] [취소]
    │     │
    │     └── 권한 영구 거부 상태 ──▶ [설정 안내 팝업] (위와 동일)
    │
    └── GPS 신호 없음 ──▶ "GPS 신호를 찾고 있습니다. 실외로 이동해주세요."
                           └── 신호 수신 시 자동으로 시작 가능 상태 전환
```

### 2.2 Patrol 진행 중 → 일시정지 → 재개

```
[U03 순찰 진행 중] (Patrol.status = 'active')
    │
    ├── GPS 좌표 수집 (5~10m 이동 또는 10초 간격)
    ├── Route(경로) 폴리라인 실시간 업데이트
    ├── 거리/시간/속도 실시간 갱신
    │
    ├── [일시정지 버튼 탭]
    │     │
    │     ▼
    │   [U04 순찰 일시정지] (Patrol.status = 'paused')
    │     ├── GPS 수집 중단
    │     ├── 타이머 정지 (duration_seconds 누적 중단)
    │     ├── 지도에 정지 구간 점선 표시
    │     │
    │     ├── [재개 버튼 탭] ──▶ [U03 순찰 진행 중] (status → 'active')
    │     │                       └── GPS 수집 재시작, 타이머 재시작
    │     │
    │     ├── [종료 버튼 탭] ──▶ 종료 확인 팝업
    │     │                       ├── "종료" ──▶ [U07 순찰 결과]
    │     │                       └── "취소" ──▶ [U04 일시정지 유지]
    │     │
    │     └── [30분 초과 정지]
    │           └── 자동 종료 안내 팝업:
    │               "30분 이상 정지 상태입니다. 순찰을 종료하시겠습니까?"
    │               ├── "종료" ──▶ [U07 순찰 결과]
    │               └── "재개" ──▶ [U03 순찰 진행 중]
    │
    ├── [마킹 버튼 탭] ──▶ (2.3 마킹 플로우)
    │
    └── [종료 버튼 탭] ──▶ 종료 확인 팝업
          ├── "종료" ──▶ [U07 순찰 결과]
          └── "취소" ──▶ [U03 순찰 진행 유지]
```

### 2.3 PatrolMark(현장 마킹) 생성

```
[U03 순찰 진행 중] ── 마킹 버튼(FAB) 탭
    │
    ▼
[U05 현장 마킹]
    │
    ├── GPS 좌표 자동 기록 (lat, lng)
    ├── 유형 선택: [일반 기록 (normal)] / [신고 (report)]
    │
    ├── [일반 기록 선택]
    │     │
    │     ├── 사진 촬영/선택 (최대 3장, 선택)
    │     │     └── 카메라 권한 미허용 시 → [U22 카메라/갤러리 권한 요청]
    │     │
    │     ├── 메모 입력 (최대 200자, 선택)
    │     │
    │     ▼
    │   [저장 버튼 탭]
    │     │
    │     ├── PatrolMark 생성 (type='normal')
    │     ├── 지도에 핀(파란색) 추가
    │     ├── 저장 완료 토스트
    │     │
    │     ▼
    │   [U03 순찰 진행 중] (복귀, 타이머 연속)
    │
    └── [신고 선택] ──▶ (2.4 신고 플로우)
```

### 2.4 Report(신고) 작성

```
[U05 현장 마킹] ── 유형 '신고' 선택
    │
    ▼
[U06 신고 작성]
    │
    ├── 카테고리 선택 (필수):
    │   ├── 불법투기
    │   ├── 시설파손
    │   ├── 안전위험
    │   └── 기타
    │
    ├── 상세 설명 입력 (필수, 최대 500자)
    │
    ├── 사진 촬영/선택 (최대 3장, 선택)
    │
    ▼
  [제출 버튼 탭]
    │
    ├── 필수 항목 미입력 ──▶ 해당 필드 에러 표시
    │
    ├── 네트워크 정상
    │     │
    │     ├── PatrolMark 생성 (type='report')
    │     ├── Report 생성 (status='submitted')
    │     ├── 지도에 신고 핀(빨간색) 추가
    │     ├── 제출 완료 토스트: "신고가 접수되었습니다"
    │     │
    │     ▼
    │   [U03 순찰 진행 중] (복귀, 타이머 연속)
    │
    └── 네트워크 오류
          │
          ├── Report 데이터 로컬 저장 (오프라인 큐)
          ├── "네트워크 연결 시 자동으로 전송됩니다" 토스트
          │
          ▼
        [U03 순찰 진행 중] (복귀)
          │
          └── [네트워크 복구 시] ──▶ 로컬 데이터 서버 전송
                                      └── 전송 완료 알림
```

### 2.5 Patrol 종료 → 결과 화면

```
[U03 또는 U04] ── 종료 확인 후
    │
    ▼
[U07 순찰 결과] (Patrol.status → 'completed')
    │
    ├── 자동 표시 항목:
    │   ├── 총 거리 (distance_km): 예) 2.35 km
    │   ├── 총 시간 (duration_seconds): 예) 00:45:12
    │   ├── 마킹 수 (PatrolMark 개수): 예) 3건
    │   ├── 신고 수 (type='report' 개수): 예) 1건
    │   └── 획득 Points (points_earned): 예) 235pt
    │
    ├── 경로 지도: 전체 Route를 축소하여 한눈에 표시
    │
    ├── 활동 일지(Patrol.note) 입력 (선택, 최대 500자)
    │
    ▼
  [저장 버튼 탭]
    │
    ├── Patrol 레코드 업데이트 (note 저장)
    ├── User.points 증가, User.total_distance_km 증가, User.total_patrols 증가
    ├── Ranking 갱신 (실시간 탭: 즉시 반영)
    ├── Points 획득 애니메이션
    │
    ▼
  [U02 순찰 대기 (홈)] 또는 [U08 순찰 기록 상세]로 이동 선택
```

### 2.6 엣지 케이스: 백그라운드 전환

```
[U03 순찰 진행 중] ── 앱 백그라운드 전환 (홈 버튼/다른 앱)
    │
    ├── GPS 수집 계속 (Foreground Service / Background Location)
    ├── 알림바 표시: "순찰 진행 중 | 거리: 1.2km | 시간: 00:23:45"
    ├── 알림바 탭 ──▶ 앱 포그라운드 복귀 → [U03 순찰 진행 중]
    │
    └── [앱 강제 종료 (kill)]
          │
          ├── 마지막 GPS 데이터까지 로컬 저장
          ├── 앱 재실행 시:
          │     │
          │     ├── 미완료 Patrol 감지 (status='active' 또는 'paused')
          │     │
          │     ▼
          │   [복구 팝업]
          │     "진행 중이던 순찰이 있습니다."
          │     ├── "이어서 시작" ──▶ [U03 순찰 진행 중] (경로 이어서 기록)
          │     └── "종료 처리" ──▶ [U07 순찰 결과] (마지막 데이터까지 결과 산출)
          │
          └── Patrol.status = 'active'인 채로 30분 경과 시
                └── 서버에서 자동으로 'completed' 처리
```

### 2.7 엣지 케이스: 배터리 부족

```
[U03 순찰 진행 중] ── 배터리 잔량 감지
    │
    ├── 배터리 20% 이하
    │     └── 경고 토스트: "배터리가 부족합니다. 순찰을 종료하는 것을 권장합니다."
    │
    ├── 배터리 10% 이하
    │     └── 경고 팝업:
    │         "배터리가 매우 부족합니다.
    │          데이터 유실 방지를 위해 순찰을 종료해주세요."
    │         ├── "종료" ──▶ [U07 순찰 결과]
    │         └── "계속" ──▶ [U03 순찰 진행 유지]
    │
    └── 배터리 5% 이하
          └── 자동 일시정지 + 로컬 데이터 저장
              └── 경고 팝업: "배터리 부족으로 순찰이 자동 정지되었습니다."
```

### 2.8 엣지 케이스: 네트워크 오류

```
[순찰 중 네트워크 연결 끊김]
    │
    ├── GPS 좌표 수집은 계속 (오프라인에서도 GPS 작동)
    ├── Route 데이터 로컬 저장
    ├── 상단 배너: "오프라인 모드 — 네트워크 연결 시 자동 동기화"
    │
    ├── PatrolMark/Report 생성 시
    │     └── 로컬 저장 → "전송 대기 중" 상태 표시
    │
    ├── [네트워크 복구 시]
    │     ├── 로컬 데이터 서버 전송 (자동)
    │     ├── 배너 해제
    │     └── "동기화 완료" 토스트
    │
    └── Patrol 종료 시 (오프라인 상태)
          ├── 결과 로컬 저장
          └── 네트워크 복구 시 서버 전송
```

## 3. Post(게시글) CRUD

### 3.1 Post 목록 조회

```
[하단 탭바: 커뮤니티]
    │
    ▼
[U10 피드]
    │
    ├── 카테고리 탭: [활동인증(activity)] [정보공유(info)] [자유(free)]
    │
    ├── 정렬: [최신순 ▼] / [인기순]
    │
    ├── Post 목록 스크롤 (무한 스크롤)
    │   └── 각 항목: 제목, 작성자 닉네임, 썸네일, 좋아요 수, 댓글 수, 작성일
    │
    ├── [Post 항목 탭] ──▶ [U11 게시글 상세]
    │
    └── [글쓰기 FAB 탭] ──▶ [U12 게시글 작성]
```
