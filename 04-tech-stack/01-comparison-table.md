### GPS 백그라운드 트래킹 관련 OS 제약사항 (공통)

| 플랫폼 | 주요 제약 | 비고 |
|--------|-----------|------|

## 3. 데이터베이스

### GPS 백그라운드 트래킹 관련 고려사항
- 순찰 경로는 연속된 GeoJSON LineString 또는 Point 배열로 저장
- 공간 쿼리: 반경 N km 내 순찰 이력 조회, 특정 구역 내 포인트 집계
- 대량 위치 데이터 적재 시 쓰기 성능 중요
- 순찰 완료 후 거리 계산, 포인트 지급은 트랜잭션 안정성 필요

| 비교 항목 | PostgreSQL (+ PostGIS) | MongoDB |
|-----------|------------------------|---------|
| **GeoJSON / 공간 쿼리 지원** | 매우 강력. PostGIS 확장으로 ST_Distance, ST_Contains, ST_Intersects 등 풍부한 공간 함수 제공. OGC 표준 준수. | 기본 지원. `$geoNear`, `$geoWithin` 등 지원하나 PostGIS 대비 공간 분석 기능 제한적. |
| **트랜잭션** | 완전한 ACID 트랜잭션. 포인트 지급, 뱃지 부여 등 복잡한 비즈니스 로직에 안정적. | 4.0 이상에서 멀티 도큐먼트 트랜잭션 지원. ACID 지원하나 PostgreSQL 대비 오버헤드 있음. |
| **확장성** | 수직 확장에 강함. 수평 확장(샤딩)은 비교적 복잡. 읽기 복제(Read Replica) 지원. | 수평 확장(샤딩)에 강함. 대용량 비정형 데이터 처리에 유리. |
| **위치 데이터 저장 효율** | 정규화된 스키마로 중복 최소화. 대량 Point 데이터는 별도 파티셔닝 권장. | 도큐먼트 기반으로 순찰 경로를 단일 도큐먼트에 내장 가능. 읽기 단순화. |
| **스키마 유연성** | 고정 스키마. 변경 시 마이그레이션 필요. | 스키마리스. 초기 개발 속도 빠름. |
| **장점** | 강력한 공간 쿼리, 완전한 ACID, SQL 표준, 성숙한 생태계, 복잡한 조인 지원 | 유연한 스키마, 수평 확장 용이, JSON 친화적, 빠른 초기 개발 |
| **단점** | 수평 확장 복잡, 스키마 변경 비용, 비정형 데이터 처리 불편 | 공간 분석 기능 제한, 복잡한 관계형 데이터 처리 불편, 강한 일관성 보장 어려움 |
| **적합 시나리오** | 복잡한 공간 쿼리 필수, 데이터 일관성 중요, 관계형 데이터 구조 명확한 경우 | 빠른 프로토타이핑, 유연한 데이터 구조, 대용량 쓰기 분산 처리 필요한 경우 |

## 5. Push 알림 — FCM + APNs 구조 설명

> FCM(Firebase Cloud Messaging)과 APNs(Apple Push Notification service)는 각각 Android와 iOS의 표준 푸시 알림 인프라입니다. 이 항목은 비교가 아닌 구조 설명입니다.

### 전체 구조

```
[서버] → [FCM (Google)] → [Android 기기]
[서버] → [APNs (Apple)] → [iOS 기기]
```

실제로 FCM은 iOS 푸시도 APNs를 통해 전달할 수 있어, 서버 측에서는 FCM 단일 연동으로 양 플랫폼 커버 가능합니다.

### 토큰 관리

| 항목 | 설명 |
|------|------|
| **디바이스 토큰 발급** | 앱 최초 실행 또는 권한 허용 시 FCM SDK가 토큰 발급 → 서버에 저장 |
| **토큰 갱신** | 앱 재설치, OS 업그레이드, 장기 미사용 시 토큰 변경됨 → 서버 갱신 로직 필요 |
| **토큰 만료 처리** | FCM 전송 실패 시 `NotRegistered` 에러 → 해당 토큰 DB에서 삭제 |
| **다중 기기 지원** | 사용자 한 명이 여러 기기 보유 가능 → 토큰을 사용자 ID에 1:N 매핑 |

### 알림 유형

| 유형 | 설명 | 순찰대 앱 활용 예시 |
|------|------|-------------------|
| **Notification 메시지** | OS가 자동으로 알림 표시. 앱이 백그라운드일 때 적합. | 관리자 공지, 순찰 완료 결과 알림 |
| **Data 메시지** | 앱이 직접 처리. 백그라운드에서도 처리 가능(제한적). | 실시간 위치 업데이트, 커스텀 알림 UI |
| **복합 메시지** | Notification + Data 조합. | 포인트 지급 알림 (배지 업데이트 + 커스텀 데이터) |

### 주의사항

- **iOS**: 푸시 알림 권한 사용자 명시적 허용 필요. 거부 시 알림 불가.
- **Android 13+**: `POST_NOTIFICATIONS` 런타임 권한 필요.
- **배경 위치 알림**: 순찰 중 Foreground Service 알림은 푸시가 아닌 OS 알림으로 표시. 푸시 알림과 혼동 주의.
- **알림 목록**: 서버 측 알림 이력 DB 저장으로 앱 내 알림함 구현 권장 (FCM은 이력 저장 안 함).
- **배치 발송 한도**: FCM HTTP v1 API 기준 단일 요청 최대 500개 토큰. 대량 발송 시 분할 처리 필요.
