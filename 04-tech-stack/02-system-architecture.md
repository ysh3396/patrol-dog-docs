│      ▼                                                      │

### 2-3. 순찰 종료 처리

```
앱: 순찰 종료 요청
        │
        ▼

        │
        ├─→ 전체 경로 좌표 집계
        │
        ├─→ 총 이동 거리 계산 (Haversine 공식 또는 공간 DB 함수)
        │
        ├─→ 순찰 통계 저장 (거리, 시간, 경로)
        │
        ├─→ 포인트 계산 및 지급 (트랜잭션 처리)
        │
        ├─→ 뱃지 달성 여부 확인 및 지급
        │
        └─→ 순찰 완료 Push 알림 발송
```

## 3. Push 알림 흐름

### 3-1. 관리자 공지 발송

```
관리자 (앱 or 웹 대시보드)
        │
        │  공지 작성 + 대상 선택
        ▼
   App Server
        │
        │  대상 사용자 FCM 토큰 조회
        ├─→ 알림 이력 DB 저장 (발송 기록)
        │
        ├─→ FCM HTTP API 호출 (Android 대상)
        │         │
        │         ▼
        │   Google FCM 서버
        │         │
        │         ▼
        │   Android 기기 수신 → 알림 표시
        │
        └─→ APNs HTTP/2 API 호출 (iOS 대상)
                  │
                  ▼
            Apple APNs 서버
                  │
                  ▼
            iOS 기기 수신 → 알림 표시
```

### 3-2. 앱 내 알림 목록

```
앱 실행 시 또는 알림 탭 진입
        │
        ▼
   App Server 알림 이력 API 조회
        │
        ▼
   DB에서 해당 사용자 알림 목록 반환
        │
        ▼
   앱 알림함 UI 표시 (읽음/안읽음 상태 포함)
```

> FCM/APNs는 전달 이력을 저장하지 않으므로, 서버 측 알림 이력 DB가 반드시 필요합니다.

## 4. 인증 흐름

### 4-1. 소셜 로그인

```
앱 → [소셜 로그인 선택]
         │
         ├─→ 카카오 OAuth 2.0
         ├─→ 네이버 OAuth 2.0
         ├─→ Google OAuth 2.0 / Sign In with Google
         └─→ Sign in with Apple (iOS 필수)
         │
         ▼
   소셜 인증 서버 → 인가 코드 반환
         │
         ▼
   App Server
         │
         ├─→ 인가 코드로 소셜 액세스 토큰 교환
         ├─→ 소셜 사용자 정보 조회 (이메일, 고유 ID 등)
         ├─→ 신규 사용자: DB 계정 생성
         │   기존 사용자: 계정 조회
         │
         ├─→ JWT Access Token 발급 (단기 만료)
         └─→ JWT Refresh Token 발급 (장기 만료, DB 저장)
         │
         ▼
   앱 → JWT 저장 (Keychain/Keystore)
         │
         ▼
   이후 API 요청: Authorization: Bearer {AccessToken}
```

### 4-2. 토큰 갱신

```
AccessToken 만료
        │
        ▼
앱 → Refresh Token으로 토큰 갱신 API 요청
        │
        ▼
서버 → Refresh Token 유효성 검증
        │
        ├─→ 유효: 새 AccessToken 발급 → 앱 저장
        └─→ 무효/만료: 재로그인 요청
```

## 5. 데이터 흐름 요약 테이블

| 기능 | 클라이언트 동작 | 서버 처리 | DB 작업 |
|------|----------------|-----------|---------|
| **순찰 시작** | GPS 수집 시작, 로컬 버퍼 초기화 | 순찰 세션 생성 | 순찰 레코드 INSERT |
| **위치 데이터 전송** | 배치 좌표 배열 POST | 유효성 검증 → GeoJSON 변환 | 위치 포인트 BULK INSERT |
| **순찰 종료** | 종료 요청 전송 | 거리 계산 → 포인트 지급 → 알림 발송 | 순찰 UPDATE, 포인트 UPDATE (트랜잭션) |
| **지도 경로 조회** | 특정 순찰 경로 요청 | 경로 좌표 조회 | 위치 이력 SELECT (순찰 ID 기준) |
| **커뮤니티 피드** | 피드 목록 요청 (페이지네이션) | 피드 데이터 조회/정렬 | 게시글 SELECT (커서/오프셋 기반) |
| **게시글 작성** | 텍스트/이미지 업로드 | 이미지 스토리지 저장 → 메타데이터 반환 | 게시글 INSERT |
| **소셜 로그인** | OAuth 인가 코드 전달 | 토큰 교환 → 사용자 조회/생성 → JWT 발급 | 사용자 UPSERT, Refresh Token 저장 |
| **푸시 알림 발송** | (수신) 알림 표시 | FCM/APNs API 호출 | 알림 이력 INSERT |
| **알림 목록 조회** | 알림함 열기 | 사용자별 알림 이력 조회 | 알림 이력 SELECT |
| **포인트 조회** | 마이페이지 진입 | 포인트 잔액 및 이력 조회 | 포인트 테이블 SELECT |
| **뱃지 조회** | 뱃지 목록 조회 | 획득/미획득 뱃지 목록 반환 | 뱃지, 사용자-뱃지 JOIN SELECT |

## 6. 주요 비기능 고려사항 (개요)

| 항목 | 고려사항 |
|------|---------|
| **보안** | 위치 데이터는 민감 개인정보 → HTTPS 필수, JWT 만료 시간 최소화, 타 사용자 위치 데이터 접근 방지 |
| **배터리** | 위치 수집 간격 조정 옵션 제공, 배경 실행 최소화, 배치 전송으로 네트워크 요청 횟수 감소 |
| **오프라인 대응** | GPS 데이터 로컬 버퍼링 → 네트워크 복구 시 자동 동기화 |
| **확장성** | 동시 다수 순찰 사용자의 위치 데이터 처리 → 오토스케일링, 데이터베이스 쓰기 최적화 |
| **개인정보** | 위치 이력 보관 기간 정책 필요, 탈퇴 시 데이터 삭제 처리, 개인정보처리방침 고지 |
